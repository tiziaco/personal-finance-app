The WelcomeCard:
- Uses `useUser()` from `@clerk/nextjs` to get `user?.firstName` (fallback: `'there'`)
- Uses `useDashboardSummary()` to get spending data
- Shows skeleton (`CardSkeleton`) while loading
- Extracts the CURRENT MONTH total expense from `data.spending.overview.stats`:
  - Sort by `month` descending (or take last element — backend orders by month asc)
  - Get the latest stat: `data?.spending.overview.stats.at(-1)`
  - Display `formatCurrency(latestStat.total_expense)` for the monthly total
- Uses shadcn `Card`, `CardHeader`, `CardTitle`, `CardDescription`, `CardContent`
- Shows the date range if available: `data?.spending.date_range.start` to `data?.spending.date_range.end`

Layout structure:
```tsx
'use client'
import { useUser } from '@clerk/nextjs'
import { useDashboardSummary } from '@/hooks/use-dashboard-summary'
import { CardSkeleton } from '@/components/shared/skeletons/card-skeleton'
import { Card, CardContent, CardHeader, CardTitle, CardDescription } from '@/components/ui/card'
import { formatCurrency } from '@/lib/format'

export function WelcomeCard() {
  const { user } = useUser()
  const { data, isLoading } = useDashboardSummary()

  if (isLoading) return <CardSkeleton />

  const latestStat = data?.spending.overview.stats.at(-1)
  const totalSpent = latestStat?.total_expense ?? 0

  return (
    <Card>
      <CardHeader>
        <CardTitle className="text-2xl">
          Welcome back, {user?.firstName ?? 'there'}
        </CardTitle>
        <CardDescription>
          {latestStat?.month
            ? `Your spending for ${latestStat.month}`
            : 'Your financial overview'}
        </CardDescription>
      </CardHeader>
      <CardContent>
        <p className="text-4xl font-bold text-primary">{formatCurrency(totalSpent)}</p>
        <p className="text-sm text-muted-foreground mt-1">spent this month</p>
      </CardContent>
    </Card>
  )
}
```

The SummaryCards renders 4 metric cards in a responsive grid (4 cols desktop, 2 cols tablet, 1 col mobile: `grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-4 gap-4`).

**Data sources (all from `useDashboardSummary()`):**

1. **Total Spent This Month** — `data.spending.overview.stats.at(-1)?.total_expense ?? 0`
   - Color: always `text-primary` (neutral — it's the baseline metric)
   - Icon: `CreditCard` from lucide-react

2. **Recurring Costs** — `data.recurring.insights.total_recurring_percentage`%  of monthly spend
   - Show as: `formatCurrency(totalSpent * (recurringPct / 100))` + label "monthly"
   - Color: `text-destructive` if recurring > 40% of income, else `text-primary`
   - Icon: `RefreshCw` from lucide-react

3. **Net Balance (Cash Flow)** — `data.spending.overview.stats.at(-1)?.net ?? 0`
   - Computed on backend as `total_income - total_expense`
   - Color: `text-destructive` if net < 0 (spending more than earned), else `text-primary`
   - Label: "Net this month"
   - Icon: `TrendingUp` or `TrendingDown` based on value (import both from lucide-react, pick conditionally)

4. **Savings** — `data.spending.recent_trend.last_3_months` average net
   - Sum of last 3 months' `net` / 3 → average monthly savings
   - Color: `text-destructive` if average < 0, else `text-primary`
   - Label: "Avg monthly savings (3mo)"
   - Icon: `PiggyBank` from lucide-react

**Note on Remaining Budget / Savings Goal (Research open question resolved):** No budget API exists (STATE.md: "Budgets page is placeholder only — no backend budget API available"). Use Net Balance and 3-month avg savings as described above — these map directly to available backend data.

**Color pattern (from STATE.md + Research):** Use `cn()` from `@/lib/utils` with `text-destructive` for overspend/negative and `text-primary` for healthy. Do NOT use inline hex colors or `text-green-*` / `text-red-*` utility classes.
