Create `web-app/src/components/dashboard/recent-transactions.tsx`.

This component:
1. Calls `useTransactions({ sort_by: 'date', sort_order: 'desc', limit: 10 }, 0)`
   - Note: The hook spreads filters into the fetch call including limit. Pass `limit: 10` to get 10 rows instead of the default 25.
2. Shows `<TableSkeleton rows={10} columns={4} />` while loading
3. Shows an empty state if `data?.items.length === 0`: a Card with "No transactions yet" and an "Upload CSV" Link to `/upload`
4. Renders a shadcn `Card` with:
   - CardHeader: title "Recent Transactions" + a `next/link` Button variant="ghost" to `/transactions` labeled "View All →"
   - CardContent: a simple list (not a full table — that's Phase 3) of up to 10 transactions

Each transaction row (use a `div` with `flex justify-between items-center py-2 border-b last:border-0`):
- Left side: merchant name (font-medium) + date below it (text-xs text-muted-foreground, use formatDate)
- Center: a Badge with the category (use shadcn Badge, variant="outline")
- Right: amount formatted with formatCurrency(item.amount) — amount is a string, formatCurrency handles it

```tsx
'use client'

import Link from 'next/link'
import { useTransactions } from '@/hooks/use-transactions'
import { TableSkeleton } from '@/components/shared/skeletons/table-skeleton'
import { Card, CardContent, CardHeader, CardTitle } from '@/components/ui/card'
import { Badge } from '@/components/ui/badge'
import { Button } from '@/components/ui/button'
import { formatCurrency, formatDate } from '@/lib/format'

export function RecentTransactions() {
  const { data, isLoading } = useTransactions({ sort_by: 'date', sort_order: 'desc', limit: 10 }, 0)

  if (isLoading) return <TableSkeleton rows={10} columns={4} />

  if (!data?.items.length) {
    return (
      <Card>
        <CardHeader><CardTitle>Recent Transactions</CardTitle></CardHeader>
        <CardContent className="flex flex-col items-center justify-center py-8 gap-3">
          <p className="text-sm text-muted-foreground">No transactions yet</p>
          <Button asChild variant="outline" size="sm">
            <Link href="/upload">Upload CSV</Link>
          </Button>
        </CardContent>
      </Card>
    )
  }

  return (
    <Card>
      <CardHeader className="flex flex-row items-center justify-between">
        <CardTitle>Recent Transactions</CardTitle>
        <Button asChild variant="ghost" size="sm">
          <Link href="/transactions">View All →</Link>
        </Button>
      </CardHeader>
      <CardContent>
        {data.items.map((item) => (
          <div key={item.id} className="flex justify-between items-center py-2 border-b last:border-0">
            <div className="flex-1 min-w-0 mr-4">
              <p className="font-medium truncate">{item.merchant}</p>
              <p className="text-xs text-muted-foreground">{formatDate(item.date)}</p>
            </div>
            <Badge variant="outline" className="mr-4 shrink-0">{item.category}</Badge>
            <span className="font-semibold shrink-0">{formatCurrency(item.amount)}</span>
          </div>
        ))}
      </CardContent>
    </Card>
  )
}
```

This component satisfies DASH-06 fully: insights callout with 1-2 highlights, "Generate New Insights" CTA in both states, and a "New data" badge when transaction data is newer than the last insights generation.

**Data sources:**
1. `useInsights()` — provides `data.insights` and `data.generated_at`
2. `useTransactions({ sort_by: 'date', sort_order: 'desc', limit: 1 }, 0)` — provides the most recent transaction date for stale-data comparison (limit: 1 is cheap, one row only)

Show `<InsightCardSkeleton />` while either hook is loading (`isLoadingInsights || isLoadingTxn`).

**Stale-data badge logic (DASH-06):**
```ts
const isStale =
  !!insightsData?.generated_at &&
  !!txnData?.items?.[0]?.date &&
  new Date(txnData.items[0].date) > new Date(insightsData.generated_at)
```
When `isStale` is true, show a shadcn `Badge` variant="destructive" with text "New data" in the `CardHeader` next to the title. This signals the user their insights don't reflect the latest uploads.

**Structure:**

```tsx
'use client'

import Link from 'next/link'
import { useInsights } from '@/hooks/use-insights'
import { useTransactions } from '@/hooks/use-transactions'
import { InsightCardSkeleton } from '@/components/shared/skeletons/insight-card-skeleton'
import { Card, CardContent, CardHeader, CardTitle } from '@/components/ui/card'
import { Badge } from '@/components/ui/badge'
import { Button } from '@/components/ui/button'
import { cn } from '@/lib/utils'

export function InsightsCallout() {
  const { data: insightsData, isLoading: isLoadingInsights } = useInsights()
  const { data: txnData, isLoading: isLoadingTxn } = useTransactions(
    { sort_by: 'date', sort_order: 'desc', limit: 1 },
    0
  )

  if (isLoadingInsights || isLoadingTxn) return <InsightCardSkeleton />

  const topInsights = insightsData?.insights?.slice(0, 2) ?? []

  const isStale =
    !!insightsData?.generated_at &&
    !!txnData?.items?.[0]?.date &&
    new Date(txnData.items[0].date) > new Date(insightsData.generated_at)

  return (
    <Card>
      <CardHeader className="flex flex-row items-center justify-between gap-2">
        <CardTitle>AI Insights</CardTitle>
        {isStale && (
          <Badge variant="destructive" className="text-xs shrink-0">New data</Badge>
        )}
      </CardHeader>
      <CardContent className="space-y-4">
        {topInsights.length === 0 ? (
          <div className="text-center py-4">
            <p className="text-sm text-muted-foreground mb-3">
              No insights yet. Upload a CSV to generate your first AI analysis.
            </p>
            <Button asChild variant="default" size="sm">
              <Link href="/stats">Generate New Insights</Link>
            </Button>
          </div>
        ) : (
          <>
            <div className="space-y-2">
              {topInsights.map((insight) => (
                <div key={insight.insight_id} className="bg-muted rounded-lg p-3">
                  <div className="flex items-start justify-between gap-2 mb-1">
                    <Badge variant="secondary" className="text-xs shrink-0">{insight.type}</Badge>
                    <span className={cn(
                      'text-xs',
                      ['high', 'critical'].includes(insight.severity)
                        ? 'text-destructive'
                        : 'text-muted-foreground'
                    )}>
                      {insight.severity}
                    </span>
                  </div>
                  <p className="text-sm font-medium">{insight.summary}</p>
                </div>
              ))}
            </div>
            {/* Generate New Insights CTA — always visible in non-empty state (DASH-06) */}
            <Button asChild variant="outline" size="sm" className="w-full">
              <Link href="/stats">Generate New Insights</Link>
            </Button>
          </>
        )}

        <div className="flex gap-3 pt-2">
          <Button asChild variant="default" size="sm">
            <Link href="/upload">Upload CSV</Link>
          </Button>
          <Button asChild variant="outline" size="sm">
            <Link href="/stats">View Insights</Link>
          </Button>
        </div>
      </CardContent>
    </Card>
  )
}
```

Named export only. No default export.

Key decisions:
- `useTransactions(..., limit: 1)` is a separate call from the RecentTransactions widget. React Query deduplicates calls with different query keys, so there is no double-fetch issue.
- `variant="destructive"` for the "New data" badge makes it visually prominent without being alarming — consistent with shadcn destructive semantics for "attention required".
- "Generate New Insights" appears in BOTH the empty state (as primary button) and the non-empty state (as outline full-width button above the CTA row) — satisfying DASH-06.
- The `isStale` check guards against `generated_at` or `items[0]` being undefined (new users with no insights or no transactions).
